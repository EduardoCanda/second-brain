O paradigma mudou para responsabilização fim a fim:

- **Se você é Dev:** Seu trabalho não acaba no _commit_ (quando você salva o código no repositório). Você precisa entender como a sua aplicação consome memória e CPU lá onde ela vai morar (no servidor/cluster).
    
- **Se você é Ops/Infra/SRE:** Seu papel não é mais barrar o Dev. É fornecer plataformas, automações e pipelines para que o Dev consiga colocar o código em produção com autonomia, mas com balizas de segurança (os _guardrails_).
    

**Resumo da Ópera:** Não existe mais "só faço código" ou "só cuido do servidor". Todos focam na saúde do produto em produção.
